## 里程碑 M2:剧情进度与碎片系统(玩法骨架)

目标:主线进度可存档、可推进。

- [ ] `ModSystem`(如 `WorldTruthSystem.cs`):存储主线剧情进度、共鸣度、玩家路线倾向(暴力/温和)
- [ ] `ModPlayer`:玩家侧数据(体内碎片觉醒度、已获能力)
- [ ] `ModItem`:记忆/残响碎片、旧 NPC 遗物(护士的手套、向导的徽章、渔夫的鱼饵盒,各带微弱异能)
- [ ] 碎片提交流程:交给 Etya → 触发歌 → 解锁副本入口
- [ ] 歌词提示 UI(`UIState`):唱歌演出的文本展示,可先做简易版(对话框 + 音效)

## 技术要点备忘

| 模块 | 用途 | tModLoader 实现 |
|------|------|-----------------|
| Etya NPC | 引导者、商店、对话 | `ModNPC`(已有骨架) |
| 剧情进度 | 主线阶段、共鸣度、结局分支 | `ModSystem`(存 world)+ `ModPlayer`(存玩家) |
| 碎片/遗物 | 收集品、副本钥匙 | `ModItem` |
| 梦境副本 | 记忆战斗场景 | 先用隔离区域 + 传送;需要独立地图时再评估子世界方案(如 SubworldLibrary) |
| 异变 Boss / 终焉者 | 剧情 Boss | `ModNPC` + `ModProjectile` |
| 歌词/剧情演出 | 唱歌 UI、真相揭示 | `UIState` + 屏幕滤镜/背景切换 |
| 隐藏 UI 战斗 | 终焉者三阶段 | `Main.hideUI` 或自定义 `IngameFadeout` 处理,需实验 |

注意事项:

- idea.txt 中提到的 `ModWorld` 是旧 API,现版本用 `ModSystem`;`SetupShop` 已被 `AddShops` 取代(现有代码已是新写法)。
- 剧情进度务必从第一天就设计好存档结构(`SaveWorldData`/`SaveData`),后补很痛苦。
- 先做垂直切片(M3)验证核心循环好不好玩,再铺内容量。
